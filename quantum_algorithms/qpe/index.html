The Quantum Phase Estimation Algorithm

If you have just learnt about the quantum Fourier transform (QFT), then learning about the quantum phase estimation (QPE) algorithm is naturally the next step. It might not be obvious as to how QPE can be useful, but it plays a key role in other algorithms, including Shor’s famous factoring algorithm.

Beware

As mentioned in the quantum Fourier transform section, many resources have their least significant qubits at the bottom. In this site we position ours at the top so the circuits we produce work with Quirk. Just bear in mind that other resources might organise their qubits in the reverse order (or ‘upside down’).

The Problem

The problem to be solved is as follows: We have a unitary matrix U, if we perform U on some qubits in one of U’s eigenstates, the qubits will gain a phase factor. Our aim is to find this phase to an acceptable accuracy.

It turns out (surprise!) that a quantum computer can do this efficiently.

The Algorithm

There are two parts to this algorithm, the first creates a Fourier state |~phi^2n⟩ and the second part uses the inverse-QFT to turn this into a classical number state we can measure and read. Below is a brief outline of the algorithm; don’t worry about understanding it all just yet, we will explain it in this section.

If you are familiar with the upside-down-CNOT identity, you will remember that the target qubit is in an eigenstate of the X-gate, and if that eigenstate is |-⟩ , the control qubit will inherit a negative phase and switch from |+⟩ to |-⟩ (or vice versa).

If this is not clear to you, I highly recommend following through with the maths to convince yourself of this. You may find reading over the upside-down-CNOT identity helpful.

There are two registers in this algorithm, one register (A) is set to the appropriate eigenstate, the other (B) is set to |+⟩ qubits. We then perform controlled-U operations on the register A, controlled by the qubits in B. Cleverly, we use multiple controlled-U operations per qubit to gain more precision.

When doing the QFT on n qubits, we can take a classical number state |x⟩ where x is between 0 and 2n, and transform it into it’s corresponding quantum Fourier state. In the QFT section we noticed a relation between this Fourier state and the number x:

Building a Quantum Circuit

Next, lets try and create a quantum circuit that carries this operation out. We’ll start with three qubits, then generalise to any number of qubits.

Firstly, its easier to implement the QFT upside down, the reasons for this will become apparent as we create the circuit but until then we will put some swaps at the end of the transform to arrange the qubits in the correct order.

Approximate QFT

You might notice that for large circuits with many qubits, we are spending a lot of time doing tiny rotations. These tiny rotations have a tiny impact on the final state of the qubits and it turns out that we can ignore rotations below a certain threshold and still get decent results. Cutting out these gates naturally improves performance and gives us a simple approximate QFT. Approximate QFTs can be created with O(nlog(n)) gates, we will not cover this here but it is worth remembering.
